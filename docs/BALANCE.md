# バランス記録

## 検証手法

ヘッドレス自動対戦シミュレータ(`src/sim.ts`)でボット同士(Lv6)の3分マッチを全キャラ総当たり。
コンソールから実行: `__tw.sim('renji','jin')` / `__tw.simMatrix(6).then(r => __tw.simSummary(r))`

**注意: ボット自己対戦は人間プレイの近似でしかない**(人間は機動・エイム・読みで近距離型を活かせる)。
シムは「構造的な歪み」の検出に使い、最終調整は実プレイのフィードバックで行う。

   (超近距離型は威力80%圏まで我慢)

## 2026-06-14 v0.3 大改修(スフィア占領モード)バランス所見

撃破数アタックを廃し**スフィア占領(陣取り)**へ全面変更。トークンも対人戦闘力を排して
盤面制圧の道具へ刷新。上記の旧キル指標(K/D・勝ち点)は占領モードでは無効。正典:
`src/objectives.ts`(占領機構) / `src/tokens.ts`(トークン) / `docs/GAME_DESIGN.md`(ルール)。

### 占領機構の実挙動(直接ステップ計測で確認)
`CAPTURE_PER_DAMAGE=0.0025`, `CAP_THRESHOLD=0.55` で `Sphere` をステップ実行:

| 項目 | 実測 | 設計意図 | 判定 |
|---|---|---|---|
| 将の射撃で中央占領(0→0.55) | 約4〜5秒 | 瞬間占領を防ぐバッファ | ✅ |
| ガンナー1機の供給(0.03/s)×5秒 | charge 0.15(未占領) | 単独は遅い=補助 | ✅ |
| 両軍が同量で撃ち合い | charge 0 維持(係争) | 相殺で拮抗・停止 | ✅ |
| 敵陣奪取(-1→+0.55) | 約13秒(中央の約2.6倍) | 敵陣は約2倍重い | ✅ |

敵陣の重さは明示倍率でなく「敵陣が満タン -1 から始まる」ことで自然発生。さらに奪取直後の側へ
3秒・供給半減ペナルティが乗る。**瞬間占領なし/相殺で膠着/敵陣は重い**の3要件を満たす。

### 要観察(試遊で重点確認)
- ガンナー×3+ヒーラー+ブースターで占領が速すぎないか(逓減で抑制済だが要実測)。
- マイン×3(最安)の経路封鎖が固すぎないか(振切り3秒で緩和済)。
- セントリー迎撃が強すぎて占領トークンが何も通らない停滞にならないか。

### 計測の制約(重要)
- ヘッドレスsimはほぼ実時間。1ステップ ~13ms(headless化で約35%短縮)、終盤はユニット増で更に重い。
- ブラウザ単発evalの上限(~30秒)内で回せるのは**ゲーム内 約25〜30秒まで**。フル3分マッチ(≒68秒実時間)・
  8キャラ総当たり(≒30分超)は**この環境では自動実行不可**。Node実行も canvas 依存で不可。
- → **最終バランスは実機試遊**で詰める。自動マトリクスが要るなら sim の WebWorker 化(canvas依存の分離)が前提。
