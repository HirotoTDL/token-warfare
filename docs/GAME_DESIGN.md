## ★v0.3 大改修: スフィア占領モード(陣取り)  ← 現行の最優先仕様

撃破数アタックを廃し、**陣取り(スフィア占領)**へ全面変更する。

### 勝敗
- フィールドに**スフィアが3つ**: **中央スフィア**(初期=無色)、**青陣スフィア**(初期=青)、**赤陣スフィア**(初期=赤)。
- **中央スフィア＋敵陣スフィアの両方**を自陣色にしている間、自軍の**カウントが加算**される(秒進行)。
- 先に**30カウント**到達で勝利。3分以内に未到達なら**カウントの多い側が勝利**(同点はサドンデス)。

### スフィア占領機構(charge メーター)
- 各スフィアは `charge ∈ [-1, +1]`。**青の攻撃で +、赤の攻撃で −**。被ダメージ量に比例して変化し、**両軍の攻撃は相殺**(同じスフィアを撃ち合えば差分で動く)。
- 色判定: `charge ≥ +CAP_TH → 青` / `charge ≤ −CAP_TH → 赤` / 中間 → 無色。
- 中央は charge=0(無色)開始、各陣スフィアは ±1(自陣色満タン)開始。
- **無色→自陣色**は 0→+CAP_TH の距離、**敵陣色→自陣色**は −1→+CAP_TH の距離 ≒ **約2倍のダメージ**が必要(ルール要件を満たす)。
- 占領後も撃ち返されれば中和・再反転する。スフィアにはHPでなく charge を使う(死なない/常設)。
- 与ダメ源: プレイヤーの武器・爆発、および一部の**占領系トークン**。

### トークン設計方針(全面刷新)
- **プレイヤー級の対人戦闘力を持たせない**。トークンは**盤面形成・戦闘補助**に寄せる。
- 戦闘系トークンは**近くの敵(将・敵トークン)へ向かい交戦**する。よって**トークン同士も戦う**。個の火力は低い。
- ロール: **索敵 / 突撃(盤面押上げ) / お邪魔(妨害) / 占領支援 / 防衛・支援**。
- 各トークン・各スキルは「**どの戦略で使うか**」を明記して設計する(下表)。

### スキルの戦略(キャラ個性は維持しつつ占領モード向けに再定義)

- ガロ/バリアドーム: 占領中の被弾を耐え押し切る「居座り」。

## コンセプト

- **PvP FPS × トークン配備ストラテジー × 3分スコアアタック**。
- プレイヤーは「将」=**シェイプリンガー**(自分の体を分裂させて武装トークンを生む人型モンスター)。
- **3分間で敵将を何度倒せるかで勝負**。倒したら終わりではなく、リスポーンして盤面の攻防が続く。
- 武器 × キャラスキル × トークン構成 × フィールド相性 = 選択個性。盤面(トークン)と本体(FPS)の二層戦略。
- 雰囲気は**パーティー感のあるわちゃわちゃ対戦**。キーワード: **パンク、ストリート、ビビット、ポップ、かっこいい、親しみ**。

## 世界観

- 武器はゴリゴリの鉄ではなく**ポップアートのようなエネルギーブラスター**。放たれるのは光る**エネルギー弾**。
- コマンダー(将)は男女8人のシェイプリンガー。**親しみのある可愛さ×洗練されたポップカルチャー/アート感**。
- トークンは将の「分裂体」。フィールドに落ちる**エナジーコア**は分裂の燃料。
- ステージはストリート感のある浮遊アリーナ(ネオン、グラフィティ、ビビットな配色)。

## コアルール(v0.2)

### マッチ
- **3分間**。敵将撃破=1スコア。**スコアの多い側が勝ち**。
- 同点で時間切れの場合は**サドンデス**(45秒延長、次の撃破で即決着。決着しなければDRAW)。
- 撃破された将は**4秒後に自基地でリスポーン**(リスポーン後2.5秒無敵)。
- 将が倒されても**配備済みトークンは消えない**(盤面全壊はさせない)。

### エネルギー(弾薬システム)
- 各武器は**エネルギー100**を共有プールとして消費。撃ち続ければ尽きる。
- **チャージ動作(Rホールド)**: +55/s回復。ただし**移動速度35%・射撃不可=無防備**。どこで回復するかが立ち回り。
- 非射撃1.5秒後から微量パッシブ回復(+7/s)。
- **連射性とエネルギー効率(燃費)は武器ごとに別**。バランスの最重要パラメータ。

### エネルギー弾
- 全ての射撃は**弾速のある発光エネルギー弾**(視認できる弾道)。
- **距離によって威力が変化**(武器ごとの距離カーブ)。近距離型/中距離型/遠距離型の住み分けの根幹。

### TP(トークンポイント)とエナジーコア
- トークン配備はTP消費。**TPの主な獲得源はフィールドのエナジーコア回収**(接触で+20TP)。
- コアは固定8スポットから抽選で湧く(同時最大4個、約12秒間隔)。**取りに行く行動=盤面リスク**。
- 敵トークンを撃破すると、その場に小コア(+10TP)がドロップ。回収には前に出る必要がある。
- TP自然回復はごく微量(+1.2/s)。コアを取らない将は分裂できない。

### ミニマップ(戦況把握)
- 画面右上に常時表示。**両軍トークンの位置は全て見える**(トークン=盤面の駒であり情報網)。
- 障害物シルエット、コア位置、自分の位置と向きも表示。
- **敵将の位置は原則表示されない**。例外(後述のソナー/チェイサー)で一時表示される場合、**表示された側には警告音とHUD通知**を出す(知らないうちに抜かれる興ざめを防ぐ)。

## 逆転設計(カムバックデザイン)

盤面ゲーは一度倒されると不利が雪だるまになりやすい。**パーティー感のあるわちゃわちゃ**を保つため、逆転の芽を仕込む。ただし:

> **原則: 逆転要素は「無料の追い風」ではなく「再建コストの割引」と「山場の演出」。打開の最後の一押しは必ずプレイヤーの操作で達成させる。あからさまなチートスキル・自動逆転は置かない(打開される側が興ざめするため)。**

1. **盤面保全**: 将が倒れてもトークンは残る。負けても盤面資産は守れる。
2. **勝者に過剰報酬を与えない**: 敵将撃破の報酬は**スコアのみ**(TPボーナスなし)。スノーボール抑制。
3. **ビハインドブースト(モメンタム)**: スコアで負けている側はTP自然回復+60%・コア取得+5TP。地味な再建割引で、露骨な戦闘強化はしない。
4. **リスポーン保護**: 2.5秒無敵+基地スタート。狩り待ち(スポーンキャンプ)はトークンの基地防衛で迎撃可能。
5. **オーバータイム(残り30秒)**: 両者TP回復2倍・コア湧き間隔半減・リスポーン2秒に短縮。終盤に全員の手数が増え、逆転の山場が自然発生する(両者に効くのでチートではない)。
6. **禁止事項(明文化)**: 全体マップワイプ、無条件即死、操作不要の自動逆転、長時間の完全無敵は実装しない。

## 練習モード

- モード選択: **対戦(vs CPU)** / **練習(レベル1〜10)**。一人プレイ前提。
- レベルはボットの「精度・反応・判断間隔・配備頻度・経済効率・偏差射撃」をスケーリング。対戦モードはLv6相当。
- Lv1: 棒立ちに近い的当て練習 ←→ Lv10: 偏差射撃+最速思考+経済ブースト。

- 生成物の品質基準: イメージワード(パンク/ストリート/ビビット/ポップ/かっこいい/親しみ)との合致、ゲーム内での視認性、チームカラーの明確さ。
- **グリッドシート(複数画像を1枚に並べる)方式は禁止。1枚ずつ個別生成**(ユーザー方針)。

## 操作

| 入力 | 動作 |
|---|---|
| WASD / Space / Shift | 移動 / ジャンプ / スプリント |
| マウス / 左クリック / 右クリック | 視点 / 射撃 / エイム |
| **R(ホールド)** | **エネルギーチャージ(無防備)** |
| E | キャラクタースキル |
| 1〜4 | トークン配備(照準先に設置) |
| M | ミニマップ拡大(予定) |

## 技術構成

- Three.js + Vite + TypeScript。モデル・SE・テクスチャは現状プロシージャル(codex製アセットに順次差し替え)。
- `World`(ユニット登録/ナビグリッド/LOS/コア/リビール状態)+ `Unit` インターフェース。ネット同期を見据えた状態分離。
- ボット難易度は `bot.ts` のレベルテーブル1箇所に集約。

## バランス指針

- 武器は「瞬間DPS/持続DPS/燃費/有効距離」の4軸で個性化し、総合点を揃える。
- トークンは「単体では将に勝てないが、放置すると盤面を失う」。
- 撃破スコアの期待値: 互角同士で3分2〜4キル程度を想定(1キルの重みと逆転可能性の両立)。
- 逆転設計の原則(上記)を破る調整はしない。
