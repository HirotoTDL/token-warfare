# TOKEN WARFARE(仮)

PvP FPS × トークン配備ストラテジー × 3分スコアアタック。
将(シェイプリンガー)を多く撃破した側が勝つ。設計の正典は [docs/GAME_DESIGN.md](docs/GAME_DESIGN.md)。

## 起動

```
npm install
npm run dev
```

→ http://localhost:5180

## 現状(v0.2)

- **3分マッチ**: 敵将撃破数で勝負。撃破されてもリスポーンして盤面の攻防が続く
- **キャラ8人**(男4女4のシェイプリンガー)+ 標準トークン3種 + キャラ固有トークン8種
- **エネルギー弾**: 距離威力カーブ・武器別ステータス・連射制限・チャージ動作(R長押し、無防備)
- **エナジーコア**: TPの主獲得源。フィールド回収で盤面リスクと配備を両立
- **ミニマップ**: 両軍トークン表示。敵将は原則非表示(ソナー/チェイサーで例外、被捕捉側に警告)
- **逆転設計**: モメンタム(ビハインドTPブースト)、オーバータイム、盤面保全(死んでもトークンは残る)
- **モード**: 対戦(Lv6) / 練習(Lv1〜10)
- ビジュアル: パンク/ストリート/ビビット/ポップ。モデル・SE・テクスチャは全てプロシージャル生成

## アート制作

- ロゴ/キービジュアル/キャラ絵/テクスチャはcodex経由でGPT生成予定(**codex CLI未導入**)
- 発注書: [x/ART_BRIEF.md](x/ART_BRIEF.md)(導入後そのまま流せる)

## 開発メモ

- コンソールに `__tw` デバッグフック: `__tw.battle`(内部状態)、`__tw.startBattle('renji')`、`__tw.setMode(レベル, true)`
- メインループは例外を `[loop error]` で出して継続
- ボット難易度は `bot.ts` の `botParams()` 1箇所に集約
- バランスの主戦場: `types.ts` の武器テーブル(威力/連射/燃費/弾速/距離カーブ)
